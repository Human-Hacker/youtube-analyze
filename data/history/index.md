# 分析履歴インデックス

## ディレクトリ構造

```
data/
├── input/      # 入力データ（不変）
│   ├── videos/             # YouTube API取得データ
│   ├── scripts/            # 台本テキスト
│   ├── video_index.json    # 動画一覧マスタ
│   ├── human_scores.json   # 人間評価スコア
│   └── analysis_fundamentals.json
│
├── output/     # 現バージョンの全出力ファイル
│   ├── golden_theory.json      # 黄金法則
│   ├── insights.md             # 仮説ログ
│   ├── analysis_conclusion.md  # 分析結論
│   ├── analysis_report.md      # 詳細分析レポート
│   ├── model.json              # 予測モデル
│   ├── integrated_analysis.md  # 統合分析
│   ├── next_26_artists.md      # 次期アーティスト選定
│   ├── g7_topicality.md        # G7実測データ
│   ├── g_st_streaming.md       # G_ST実測データ
│   ├── g_yt_ecosystem.md       # G_YT実測データ
│   └── ...                     # その他サイクルで生成されたファイル
│
└── history/    # 過去バージョンのスナップショット
    ├── index.md            # このファイル（変更ログ）
    ├── v5.9_20260216/      # output/の全ファイルコピー
    └── v6.0_20260216/
```

### 各フォルダの役割

| フォルダ | 役割 | 更新タイミング |
|---------|------|--------------|
| `input/` | 不変の入力データ | 新動画追加時のみ |
| `output/` | 現バージョンの全出力（常にここを参照） | 分析サイクルごと |
| `history/` | 過去バージョンのスナップショット保管庫 | output更新後にコピー |

### バージョン管理ルール

1. 分析サイクル完了時に `output/` の全ファイルを `history/vX.X_YYYYMMDD/` にコピー
2. 各スナップショットフォルダにはその時点の output/ ファイル全てを含める
3. `index.md` にバージョン番号・日付・サマリを追記
