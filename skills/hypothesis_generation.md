# 仮説生成方法論（Hypothesis Generation Methodology）

> このファイルは Step 9（仮説生成エージェント）専用のドメイン知識である。
> 4軸モデルの因果仮説を生成するための方法論と理論的基盤を提供する。

---

## 0. 第一原理: YouTube動画が伸びる公式

```
Views = Impressions × CTR × f(Retention)
```

- **Impressions**: アルゴリズムが動画を表示した回数（制作者はコントロール不可）
- **CTR**: クリック率。サムネイル×タイトル×トピック×タイミングで決まる
- **f(Retention)**: 維持率がアルゴリズムに与える正のフィードバック。維持率が高い→推薦が増える→Impressionsが増える

つまり: **CTRで人を呼び、Retentionでアルゴリズムを味方につける**。この2つが揃わなければ伸びない。

## 1. 仮説生成方法論

### 1.1 水平思考（Lateral Thinking）
仮説生成の第一段階。既存の前提を疑い、新しい視点を獲得する。

**手順**:
1. Phase 2-4の発見から暗黙の前提を列挙する
2. 各前提を「もし逆だったら？」で反転させる
3. 反転した前提から導かれる仮説を検討する
4. 最も反証可能で実用的な仮説を選ぶ

**例**:
- 前提: 「GIが高いアーティストの動画は伸びる」
- 反転: 「GIが高くてもMISSになる条件は？」
- 仮説: 「GIが高くてもCA（好奇心一致度）が低ければMISSになる」

### 1.2 第一原理分解（First Principles Decomposition）
公式の各項を分解し、各因子の寄与を個別に分析する。

```
Views = Impressions × CTR × f(Retention)
      ↓
Impressions ← f(Day1_CTR, Day1_Retention, チャンネル規模)
CTR ← g(Topic_Power, Hook_Power, タイミング)
f(Retention) ← h(台本構造, 感情曲線, MV配置, 導入30秒)
```

各因子のうち「制作者がコントロール可能」なもの（原因指標）だけを仮説の変数として使用する。

### 1.3 仮説駆動分析（Hypothesis-Driven Analysis）
データから帰納的にパターンを見つけるのではなく、まず仮説を立てて演繹的に検証する。

**手順**:
1. 中心的な問いに対する仮の回答（仮説）を立てる
2. 「この仮説が正しいなら、全24本でこうなるはず」を予測する
3. 予測とデータを照合する
4. 不一致があれば仮説を修正する

## 2. 反証可能性要件

### 2.1 良い仮説の条件
全ての仮説は以下の条件を満たすこと:

| 条件 | 説明 |
|------|------|
| 具体的 | 曖昧な表現を避け、測定可能な変数を使う |
| 反証可能 | 「何が起これば仮説が誤りと証明されるか」が明確 |
| 実用的 | 仮説が正しければ「次の動画企画にどう活かせるか」が示せる |
| 原因指標のみ | 結果指標を予測因子として使わない |
| 検証条件付き | 「24本中何本で成立すれば正しいとするか」の閾値を設定 |

### 2.2 仮説フォーマット
```
H{N}: {仮説名}
- 仮説: {1行で}
- 対象軸: {Topic / Hook / Retention / Virality / 複合}
- 使用変数: {この仮説で使う変数。「原因指標」か「結果指標」かを明記}
- 根拠: {Phase 2-4のどのデータから導いたか}
- 予想適用精度: {24本中何本で成立するか}
- 検証方法: {どうすれば確認/棄却できるか}
- 反例候補: {この仮説と矛盾しそうなデータ}
- 実用的価値: {この仮説が正しい場合、次の動画企画にどう活かせるか}
- 感度分析: {閾値±2段階での精度変化}
```

### 2.3 禁止される仮説タイプ
- **結果指標仮説**: 「エンゲージメント率が高い動画はHITになる」（← 結果が原因を説明している）
- **トートロジー仮説**: 「再生数が多い動画は伸びる」（← 定義の反復）
- **漠然仮説**: 「良い台本の動画は伸びる」（← 測定不能）
- **棄却済み再提案**: insights.mdの棄却仮説と同じ方向の仮説
- **AI GI/CA使用**: AI評価のGI/CAスコアを予測因子に使った仮説

## 3. N=24の統計推論

### 3.1 N=24の限界
- 統計的有意性の閾値（p < 0.05等）を満たすことは困難
- 相関係数の信頼区間が広く、相関分析の信頼性が低い
- 複数の変数を同時に制御する多変量分析は不可能

### 3.2 N=24で有効な手法
| 手法 | 説明 |
|------|------|
| パターンの一貫性 | 「全HIT動画でX、全MISS動画で非X」のような完全分離を探す |
| 閾値分析 | 「X以上ならHIT確率が高い」のような閾値を探す |
| 感度分析 | 閾値を±2段階で変化させたときの精度変化を確認 |
| ケースコントロール | 条件が似たペアの差分を分析（Phase 4で実施済み） |

### 3.3 推論の強度基準
| 強度 | 条件 | 例 |
|------|------|-----|
| 強い | 全HIT/全MISSで例外なし（24/24） | 「GI×CA > 40の動画は全てHIT」 |
| やや強い | 1-2本の例外あり（22/24以上） | 例外に合理的説明があれば「条件付き」で採用 |
| 中程度 | 3-6本の例外（18/24以上） | 追加条件で修正可能かを検討 |
| 弱い | 7本以上の例外（17/24以下） | 仮説として不十分、再構成が必要 |

## 4. 4軸モデル理論

### 4.1 4軸の定義
| 軸 | 意味 | 主な原因指標 |
|----|------|-------------|
| Topic Power | 「この人の話を聞きたい」力 | GI, 時事性, 競合密度 |
| Hook Power | 「このサムネ/タイトルをクリックしたい」力 | CA, タイトル設計 |
| Retention Power | 「最後まで見たい」力 | 台本構造, 感情曲線, MV配置, 導入30秒 |
| Virality Power | 「誰かに教えたい」力 | （制作者のコントロール困難。観測が中心） |

### 4.2 軸の独立性
- 各軸は独立した因果経路を持つ
- Topic Powerが高くてもHook Powerが低ければCTRが低い
- Hook Powerが高くてもRetention Powerが低ければ推薦されない
- 「どの軸が不足してMISSになったか」を動画ごとに特定する

### 4.3 成功パターンの類型化
- **全軸高型**: 確実にHIT
- **Topic+Hook高、Retention低型**: CTR高いが推薦が伸びない（フック詐欺）
- **Topic高、Hook低型**: 潜在力は高いがクリックされない（隠れた良作）
- **全軸低型**: 確実にMISS

## 5. 視聴者心理の原則

### 5.1 好奇心ギャップ
- 「知りたいこと」と「現在知っていること」のギャップが視聴動機
- タイトルでギャップを作り、動画内で段階的に解消

### 5.2 感情移入と物語構造
- 視聴者は主人公の感情に同一化することで視聴を継続する
- 感情の「底」が深いほど、その後の回復に対するカタルシスが大きい

### 5.3 ゴシップ欲求
- 有名人の裏話、知られていない事実への関心は普遍的に強い

### 5.4 楽曲の力（音楽チャンネル固有）
- 知名度の高い楽曲のMV挿入は感情のピークを作れる
- 楽曲を知らない視聴者にとってはMV挿入が離脱ポイントになり得る

## 6. チャンネル固有データ（ウタヨミ 24本）

### 6.1 チャンネル概要
- **コンテンツ**: 海外音楽アーティストの伝記・物語形式の長尺動画（13〜22分）
- **動画数**: 24本（2025年7月〜2026年2月）
- **登録者推移**: 約20,000人 → 約47,000人（約136%成長）

### 6.2 視聴者プロファイル
- **コア視聴者層**: 45〜64歳の女性
- **含意**: 懐かしさへの共感が視聴動機の中核

### 6.3 トラフィック構成
- **主要流入**: ブラウジング 38〜74% + 関連動画 22〜50%（合計70〜96%）
- **含意**: アルゴリズム推薦への依存度が極めて高い

### 6.4 視聴維持パターン
- **平均視聴時間**: 5〜8分（動画長の29〜39%）
- **新規視聴者**: 53〜57%（過半数が初見）

### 6.5 HIT/MISS閾値
- **HIT**: 15万回以上 / **MISS**: 15万回未満

## 7. 指標分類（原因 vs 結果）

### 7.1 原因指標（制作者がコントロール可能 = 仮説の変数として使用可）
- アーティスト選定（Topic Power）
- タイトル・サムネイル設計（Hook Power）
- 台本構造: フック設計、MV配置、感情曲線、導入30秒（Retention Power）
- 公開タイミング（曜日、時事性）

### 7.2 結果指標（伸びた結果として発生 = 仮説の変数として使用禁止）
- 再生数、IMP数、ブラウジングIMP量、登録者視聴数
- 新規視聴者率、SUBSCRIBER比率、RELATED比率
- 年齢層分布、性別比率
- Day1視聴数、Day1→Day2変化率
- エンゲージメント率、平均視聴時間、平均視聴率
