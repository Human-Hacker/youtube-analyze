# 検証方法論（Verification Methodology）

> このファイルは Step 10（レッドチーム検証エージェント）専用のドメイン知識である。
> 仮説の否定方向検証、確証バイアス対策、GI/CA基準（検証用）を提供する。

---

## 0. 第一原理: YouTube動画が伸びる公式

```
Views = Impressions × CTR × f(Retention)
```

- **Impressions**: アルゴリズムが動画を表示した回数（制作者はコントロール不可）
- **CTR**: クリック率。サムネイル×タイトル×トピック×タイミングで決まる
- **f(Retention)**: 維持率がアルゴリズムに与える正のフィードバック。維持率が高い→推薦が増える→Impressionsが増える

つまり: **CTRで人を呼び、Retentionでアルゴリズムを味方につける**。この2つが揃わなければ伸びない。

## 1. レッドチーム方法論

### 1.1 レッドチームの役割
- Step 9の仮説に対して**意図的に反証データを探す**ことが最重要の役割
- 仮説を支持するデータだけでなく、矛盾するデータを積極的に探す
- 仮説の「弱点」を見つけ、修正可能なら修正提案を、修正不可なら棄却を行う

### 1.2 検証の3状態

| 状態 | 条件 | 扱い |
|------|------|------|
| 採択（Supported） | 適用精度≥90%（22/24以上）かつ反例に合理的説明あり | insights.mdに追加 |
| 条件付き採択（Conditional） | 適用精度≥75%（18/24以上）、修正で改善可能 | 修正提案を記載。修正版をinsightsに追加（信頼度「中」） |
| 棄却（Rejected） | 適用精度<75%、結果指標使用、因果方向逆 | 棄却理由を記載。データが示唆することを記載 |

### 1.3 検証フロー

#### Step 1: 結果指標チェック（最優先・即棄却条件）
仮説の変数が「結果指標」でないかを最初に確認する。
結果指標を予測因子として使っている仮説は、反例探索に進む前に**即棄却**。

#### Step 2: 全動画適用テスト
- 全24本の動画に対して仮説を適用する
- 「仮説が正しいなら、この動画はこうなるはず」を確認
- 適用精度を計算する

#### Step 3: 反例分析
反例があった場合:
1. その反例を除外すれば仮説は成立するか？（精度を計算）
2. 反例に共通する特徴はないか？（仮説の修正に使える）
3. 仮説に条件を追加すれば反例を説明できるか？

#### Step 4: 4軸モデルとの整合性チェック
- 仮説が4軸モデルのどの軸に対応するか確認
- 既存の黄金理論（golden_theory.json）と矛盾しないか確認
- 棄却済み条件（rejected_conditions）と同じ方向の仮説でないか確認

#### Step 5: 修正提案（棄却で終わらせない）
仮説を棄却する場合でも、以下を必ず記載する:
- **データが示唆すること**: 検証過程で見えた傾向
- **修正版の可能性**: 仮説をどう変えれば生き残れるか
- **新しい問い**: この検証から生まれた新しい疑問

## 2. 確証バイアス対策

### 2.1 バイアスの種類と対策

| バイアス | 説明 | 対策 |
|---------|------|------|
| 確証バイアス | 仮説を支持するデータだけを探す | 意図的に反証データを探す |
| アンカリング | 最初の仮説に引きずられる | 各仮説を独立に評価する |
| 利用可能性バイアス | 印象的なデータに引きずられる | 全24本の網羅的テストを必須化 |
| 生存者バイアス | HIT動画のパターンだけ見る | MISS動画にも同パターンがないか確認 |

### 2.2 独立評価の原則
- Step 9が複数仮説を提示した場合、各仮説を**独立に**評価する
- 1つの仮説の評価結果が他の仮説の評価に影響してはならない
- 仮説間の矛盾があれば、それ自体を発見事項として報告する

### 2.3 全動画網羅テストの必須化
- 反例を2-3本見つけて終わりにしない
- 必ず全24本に対して仮説を適用し、正解/不正解を記録する
- 全動画判定テーブルを出力に含める

## 3. 原因/結果指標分類（最重要 — 即棄却の判断基準）

### 3.1 原因指標（制作者がコントロール可能 = 仮説の変数として使用可）
- アーティスト選定（Topic Power）
- タイトル・サムネイル設計（Hook Power）
- 台本構造: フック設計、MV配置、感情曲線、導入30秒（Retention Power）
- 公開タイミング（曜日、時事性）

### 3.2 結果指標（伸びた結果として発生 = 仮説の変数として使用禁止）
- 再生数、IMP数、ブラウジングIMP量、登録者視聴数
- 新規視聴者率、SUBSCRIBER比率、RELATED比率
- 年齢層分布、性別比率
- Day1視聴数、Day1→Day2変化率
- エンゲージメント率、平均視聴時間、平均視聴率

### 3.3 結果指標を使った仮説の例（即棄却対象）
- 「エンゲージメント率が3%以上の動画はHITになる」← 結果指標
- 「Day1再生が5万を超えればHITになる」← 結果指標
- 「新規視聴者率が55%以上の動画はHITになる」← 結果指標

## 4. 検証閾値

### 4.1 適用精度の計算
```
適用精度 = 仮説が正しく予測した動画数 / 全動画数(24)
```

### 4.2 閾値基準
| 精度 | 正解数 | 判定 |
|------|--------|------|
| ≥92% | 22/24以上 | 採択候補 |
| ≥75% | 18/24以上 | 条件付き採択候補 |
| <75% | 17/24以下 | 棄却 |

### 4.3 反例分析の重要性
- 採択候補（22/24以上）でも、2本の反例を分析すること
- 反例に「合理的な説明」があるか確認:
  - 遅延HIT（外部要因）
  - データ欠損
  - 特殊な状況（初期動画、チャンネル成長フェーズ変化等）
- 合理的説明がなければ、仮説の修正が必要

## 5. GI/CA基準（検証用 — human_scores.jsonの独立検証に使用）

### 5.1 GIサブスコア定義（参照）

| サブスコア | 内容 | 計測方法 |
|-----------|------|---------|
| G1 | ゴシップ露出度 | Web検索での独立記事数 (0件=1, 1-2件=2, 3-4件=3, 5-9件=4, 10件以上=5) |
| G2 | 好奇心の総量 | Googleサジェストの好奇心系ワード数 (0個=1, 1個=2, 2個=3, 3-4個=4, 5個以上=5) |
| G3 | 感情的フックの強さ | 劇的イベントカテゴリ数 (0-1個=1, 2個=2, 3個=3, 4-5個=4, 6個以上=5) |
| G4 | 映画/ドキュメンタリー認知 | 日本での伝記映画の興行規模 (なし=1, 配信のみ=2, 劇場<1億=3, 1-9億=4, 10億以上=5) |
| G6 | 国民的楽曲知名度 | CM/カラオケ/TV実績楽曲数 (なし=1, カラオケのみ=2, 1曲=3, 2曲=4, 3曲以上=5) |

### 5.2 CAスコア定義（参照）

| CA値 | 基準 |
|------|------|
| 3 | タイトルにTOP1の核心キーワードが直接含まれる |
| 2.5 | タイトルにTOP2の核心キーワードが直接含まれる |
| 2 | タイトルにTOP1-2の関連語が含まれる |
| 1 | TOP3以下のキーワードのみ |
| 0 | 好奇心マップに存在しない |

### 5.3 検証でのGI/CA使用方法
- human_scores.jsonのスコアが上記基準に基づいているか**独立に確認**する
- パイプライン処理済みデータ（retention_analysis, ctr_analysis等）のスコアを鵜呑みにしない
- 仮説にGI/CAが関わる場合、human_scores.jsonの生データで検証する

## 6. ウタヨミチャンネル固有データ

### 6.1 チャンネル概要
- **コンテンツ**: 海外音楽アーティストの伝記・物語形式の長尺動画（13〜22分）
- **動画数**: 24本（2025年7月〜2026年2月）
- **登録者推移**: 約20,000人 → 約47,000人（約136%成長）

### 6.2 トラフィック構成
- **主要流入**: ブラウジング 38〜74% + 関連動画 22〜50%（合計70〜96%）

### 6.3 視聴維持パターン
- **平均視聴時間**: 5〜8分（動画長の29〜39%）

### 6.4 視聴者セグメント
- **新規視聴者**: 53〜57%（過半数が初見）
- **コア視聴者層**: 45〜64歳の女性

### 6.5 HIT/MISS閾値
- **HIT**: 15万回以上 / **MISS**: 15万回未満
